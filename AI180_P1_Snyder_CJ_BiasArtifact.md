---

## Workflow Strip

```
[Open AI tool] → [Paste broken code] → [Tell AI: don't change my style]
    → [Ask AI to fix it] → [Accept both fixes] → [Ask AI to check for more issues]
        → [Did it work?] → YES: Done / NO: Loop back
```

*Trace this path before you read anything below.*

---

## Card 1 — Anchoring
**The moment:** "Open AI tool without deciding what I want it to do."

When there is no goal defined before the session starts, the AI's first output becomes the standard everything else gets measured against. You are not evaluating the work — you are evaluating whether it matches what the AI already gave you.

**Counter:** Before opening the tool, write one sentence: *What am I testing?*

---

## Card 2 — Einstellung Effect
**The moment:** "Tell AI: don't change my style."

This instruction came from experience — familiarity with my own approach felt like a reason to protect it. But it was a constraint placed before any evaluation happened. The AI never got to show whether another way was better. Past experience blocked present judgment.

**Counter:** Ask the AI for one solution that does NOT match your current approach. Look at it before deciding.

---

## Card 3 — Confirmation Bias
**The moment:** "Accept 2 different fixes for code without question."

The AI returned two different solutions. I accepted both. I wanted the problem solved — the AI said it was solved — and that was enough. I never asked which fix was actually correct, or whether either of them addressed the root cause.

**Counter:** Before accepting any output, name one thing that could still be wrong with it.

---

## Card 4 — Automation Bias
**The moment:** "Ask AI to find any other issues with code."

This looks like due diligence. It is not. It is asking the same system that introduced the problem to verify that the problem is gone. The AI cannot catch errors caused by its own blind spots. Delegating verification to the tool is not checking the work — it is trusting the tool to audit itself.

**Counter:** Run one manual check that does not involve the AI before calling the work done.

---

## Why the form works this way

The workflow strip forces you to trace the sequence before reading any card. That mirrors how the biases actually operated — each one was enabled by the last. Anchoring came first and set the frame. Einstellung locked in the constraint. Confirmation bias let the fixes through unchecked. Automation bias handed the final review back to the same tool.

You cannot see Card 4 without passing through Cards 1, 2, and 3. That is not a navigation choice. That is the argument.
